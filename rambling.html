Blurbs, a.k.a tips

Semantic CSS classes are for development and collaboration works.

Why you may ask? Or maybe what?


Here is an example of a content-oriented VS semantic classes:

          /* Content */
          .name {
             color: lightblue;
             font-size: x-large;
           }
           
           /* Semantic */
           #title {
              font-size: x-large;
           }
           
           .colour-sky {
              color: lighblue;
           }

Notice how much shorter the single, object oriented block is compared to the semantic classes, although they do the exact same thing, which is shown below.

Unless you are currently working on either a website template, or a CSS framework, there is no need to make semantic classes, specify styles to exactly which and which element to style.

Website making and hosting for absolute newbies

So, you badly need a website, fast, but don't have time to set up a website and a server? Don't you worry, I provided this lazy-ass list for you to choose from.

  1. Google Sites. It features premade templates, drag and drop placement, and the best, all of them is very responsive.
  2. GitHub Pages. If you know some technical know hows, such as designing an actual website, GH Pages. And did you know, that this website is hosted on GitHub Pages?
  3. You could search for other ready-to-start site publishing services online, many even faster to set up than my list here.
